Step-by-Step Guide to Setting Up Terraform, AWS CLI, and Your AWS Environment

Day 2 of 30 Days Challenge

Today, I read the book Terraform: Up & Running by Yevgeniy Brikman — Chapter 2.

First, I created an IAM user on the AWS console, so this user will have least permissions. I connected Terraform to my AWS IAM user. I followed these steps:

  • Open IAM service on my AWS account
  • Click Users → Create user
  • Attach policies directly: AdministratorAccess
  • Click "Create user"
  • Enable programmatic access:
    • Click on user name
    • Security credentials → Create Access key
    • Choose "Command Line Interface"
    • Create
    • Save Access Key ID and Secret Key (because you will use it on VSC to set up)

For Terraform to make changes in my AWS account, I set the AWS credentials for the IAM user I created earlier (Christ) as the environment variables AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY. I ran these two commands in the terminal: 

$ export AWS_ACCESS_KEY_ID=(My IAM user access key id)
$ export AWS_SECRET_ACCESS_KEY=(My IAM user secret access key)
Important Note: I learned that it is not a good practice to use the root user with Terraform. The root user has full privileges over my AWS account, and by mistake, it's possible to delete all my resources (S3, EC2, IAM…)
