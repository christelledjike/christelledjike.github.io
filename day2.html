Day 2 Step-by-Step Guide to Setting Up Terraform, AWS CLI, and Your AWS Environment

Today I read the book Terraform: Up & Running by Yevgeniy Brikman — Chapter 2.

Step 1: Create IAM User on AWS Console

First, I created an IAM user on the AWS console with limited permissions and connected Terraform to this IAM user. The steps I followed:

  • Open IAM services on my AWS account
  • Click Users → Create user
  • Attach policies directly: AdministratorAccess
  • Create user
  • Enable programmatic access:
    • Click on the username
    • Go to Security credentials
    • Create Access Key
    • Choose Command line interface
    • Create and save Access Key & Secret Key for use in VS Code

For Terraform to make changes in my AWS account, I set the AWS credentials for the IAM user I created earlier (Christ) as environment variables:

export AWS_ACCESS_KEY_ID=(My IAM user access key id)
export AWS_SECRET_ACCESS_KEY=(My IAM user secret access key)

Important Note: It is not a good practice to use the root user with Terraform. The root user has full privileges over the AWS account, and by mistake, it’s possible to delete all resources (S3, EC2, IAM…).
