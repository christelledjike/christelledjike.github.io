What IaC is and the problem it solves?

Infrastructure as Code (Iac) is a practice in IT where infrastructure like servers, networks, storage and databases is defined and managed using code instead of manual configuration. This practice helps It team automate tasks.

IaC solves many problems:

  • Slow deployment: Manually creating infrastructure can takes many hours, slowing development, IaC can solves this by allowing fast and repeatable deployments.
  • Manual error: Mistakes can happen when setting up or changing servers.
  • Poor scalability: manual set up does not scale well for large systems
  • Inconsistent environment: Manuel configuration can lead some differences between environments.

In conclusion, IaC makes infrastructure repeatable, manageable and consistent, It's also automate tasks.

The difference between declarative and imperative approaches:

In Terraform, there are two main approaches:

  • Imperative approach: this approach telling the system exactly how to do something, step by step. You also define the sequence of commands needed to achieve a desire result.
  • Declarative approach: this approach describe the desired end state and the system determines how to achieve it.

Why Terraform is worth learning?

Learning Terraform is worth learning, for some reasons:

  • It's one of the most popular tools.
  • It's automates tasks (creation, modification, destruction of resources) reducing manual work and human errors
  • IT's works with many cloud providers (AWS, Azure, GCP) allowing to manage infrastructure across multiple platforms with a single tool.
  • It's a declarative language
